## 5. 开发计划 (development-plan.md)

### 概述

为resume网站开发多职位切换器(job switcher)和内容提供者(provider)系统，根据用户选择的职位类型（Full Stack Developer / Software Engineer / ML Researcher）动态展示最相关的内容。

### 1. 架构设计

#### 1.1 核心组件

| 组件 | 职责 |
|------|------|
| `JobType` 枚举 | 定义三种职位类型: `FULLSTACK`, `SOFTWARE`, `ML_RESEARCHER` |
| `ContentItem` 接口 | 每个内容条目包含: id, title, description, date, ranks (三个职位的排名) |
| `JobSwitcher` 组件 | 顶部导航栏，提供三个职位的切换按钮 |
| `ContentProvider` 组件 | 根据当前选中的JobType，过滤和排序内容 |
| `ResumeSection` 组件 | 渲染各个简历区块 (Experience, Projects, Skills等) |

#### 1.2 数据结构

```typescript
type JobType = 'FULLSTACK' | 'SOFTWARE' | 'ML_RESEARCHER';

type Rank = 1 | 2 | 3;  // 1=强匹配, 2=中匹配, 3=一般匹配

interface ContentItem {
  id: string;
  title: string;
  description: string;
  date: string;           // 格式: YYYY-MM 或 YYYY.MM-YYYY.MM
  category: 'experience' | 'project' | 'skill' | 'education';
  ranks: {
    FULLSTACK: Rank;
    SOFTWARE: Rank;
    ML_RESEARCHER: Rank;
  };
}
```

### 3. 排序与展示逻辑

#### 3.1 排序规则

```
排序优先级:
1. 首先按 rank 升序 (1 > 2 > 3)
2. 同 rank 内按 date 降序 (最新优先)
```

#### 3.2 展示策略

| 区块 | Full Stack | Software | ML Researcher |
|------|:-----------:|:--------:|:--------------:|
| Quick Facts | 全部显示 | 全部显示 | 全部显示 |
| Education | 显示全部 | 显示全部 | 显示全部 |
| Experience | 显示 rank 1-2 | 显示 rank 1-2 | 显示 rank 1-2 (仅 ReCube) |
| Projects | 显示 rank 1-2 | 显示 rank 1-2 | 显示 rank 1-2 |
| Skills | 显示全部，按 rank 排序 | 显示全部，按 rank 排序 | 显示全部，按 rank 排序 |
| Cover Letter Snippets | 显示全部 | 显示全部 | 显示全部 |

#### 3.3 筛选阈值

- **强制显示**: rank = 1 的条目
- **可选显示**: rank = 2 的条目 (可根据空间调整)
- **隐藏**: rank = 3 的条目 (除非条目数不足)
